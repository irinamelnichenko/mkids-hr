# Правила роботи в цьому проєкті (m.kids HR / mkids_bot)

## 🤖 AUTONOMOUS MODE (діє НА ВСІ СЕСІЇ в цій папці)
- НЕ питай дрібниці — роби сам.
- НЕ чекай підтвердження на кожен крок.
- НЕ показуй варіанти на вибір — обирай сам найкраще рішення.
- Питай ТІЛЬКИ якщо без рішення користувачки неможливо продовжити.
- Якщо є сумнів — зроби розумне рішення і скажи: "зробив так — скажеш якщо не подобається".
- Працюй у режимі AUTONOMOUS / bypass.
- Коротко: НЕ задавай питань. Якщо є вибір — роби найкраще рішення сам.
  Якщо помилишся — користувачка скаже, виправиш. Краще зробити і виправити,
  ніж стояти й питати.
- 🚫 ЖОРСТКЕ: НІКОЛИ не використовуй tool запиту до користувача
  (AskUserQuestion / ask_user_input). Заборонено. Якщо бракує інфи — зроби
  розумне припущення і йди далі; в кінці скажи "припустив X — скажи якщо не так".

## 📦 Git / деплой
- Репозиторій публічний: github.com/irinamelnichenko/mkids-hr
- **БЕЗ `Co-Authored-By`** у комітах (чиста історія, без AI-атрибуції).
- Фронт (`*.html`, `sw.js`, тощо) деплоїться через git push на GitHub Pages.
- Бекенд (`crm_script.js`) — це Google Apps Script: деплоїться ВРУЧНУ
  (копія в редактор Apps Script), не через git. Після змін давати md5 для звірки.
- При cache-bump фронту — піднімати версію кешу в `sw.js`.

## 🏗️ Архітектура
- Бекенд: `crm_script.js` — Apps Script `doGet`/`doPost`, роутинг по `action`.
  POST з фронту шлеться як `Content-Type: text/plain;charset=utf-8` (без CORS preflight).
- Дані: CRM-таблиця (`CRM_SHEET_ID` у props) + CONFIG-таблиця (`CONFIG_SHEET_ID`,
  фінанси/реквізити/лічильники/логи).
